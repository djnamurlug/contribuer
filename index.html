Une autre manière de contribuer c'est de tester une nouvelle version

Pour ça il faut aller voir sur le site web du programme pour voir où il enregistre ses versions en developpement.

Pour l'instant c'est souvent sur des forges logicielles

Gestionnaires de versions

  • Git : développé par Linus Torvald
    • Gitlab
    • Github : bugs, site web, coté "social"
    • Framagit (Gitlab aussi)
      • guide fr autre tuto
  • Sourceforge en très grosse perte de vitesse
  • Launchpad (Ubuntu)

Quand on a téléchargé la dernière version, ça demande souvent des dépendances qui ne sont installé d'origine et des bricolages.

La meilleur des solutions c'est d'isoler sa version de test pour éviter les effets de bords

  • Virtualisation (virtualbox, docker)
  • chroot et schroot : sur ubuntu-fr, debian.org/fr

    • normalement il y avait une demo mais pas eut le temps

  • pour des programmes en python : virtualenv et Pew
